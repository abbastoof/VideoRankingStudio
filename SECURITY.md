# Security policy

## Reporting a vulnerability

If you discover a security issue, **do not open a public issue**. Email `security@videorankingstudio.local` (replace with the real address once the domain is provisioned). Include:

- A clear description of the vulnerability.
- Reproduction steps or proof-of-concept.
- The commit hash or deployed environment where you observed it.

We aim to acknowledge within 48 hours and resolve high-severity issues within 14 days.

## Supported versions

Only the `main` branch (and currently-deployed production tag) receive security updates. Older tags are not patched.

## Scope

In scope:
- The `apps/web`, `apps/api`, and `apps/workers` services and their hosted endpoints.
- The infrastructure-as-code under `infrastructure/` when applied to our environments.
- Authentication, billing, file upload, and AI-job pipelines.

Out of scope:
- Self-hosted forks running with default secrets.
- Findings that require physical access to a user's device or compromised third-party services (Stripe, AI providers).
- Reports generated by automated scanners without proof of exploitability.

## Hardening summary

See [`docs/SECURITY.md`](docs/SECURITY.md) for the detailed threat model. Highlights:

- All traffic over TLS; HSTS preload in production.
- Argon2id for OTP storage; bcrypt for any future password hashes.
- JWT secrets ≥ 32 bytes, rotated quarterly. Refresh tokens are opaque, server-stored, and rotated on use.
- HttpOnly, Secure, SameSite=Lax cookies. CSRF protection on state-changing routes via double-submit token.
- CSP with `default-src 'self'`; nonces for inline scripts where unavoidable.
- File uploads are size-bounded and MIME-sniffed (not trust user-supplied content-type). All uploaded media transits MinIO/S3, never the API filesystem.
- Workers run in an egress-restricted subnet; only the AI providers configured in env, S3, and Stripe are reachable.
- Stripe webhook signatures verified against `STRIPE_WEBHOOK_SECRET`.
- Secrets in AWS Secrets Manager (or `.env` locally). Never echoed in logs. `pino`/`structlog` redaction lists configured.
- Dependency audits run in CI: `pnpm audit`, `pip-audit`, `trivy` for container images.
- All admin actions are logged in `AuditLog` with actor, action, target, and IP.

## Responsible-use stance

VRS may not be used to:
- Impersonate a real person without consent (deepfake misuse).
- Produce content that infringes copyright, harasses, or violates applicable law.
- Bypass platform terms of service (e.g. mass-scraping YouTube).

Voice cloning requires affirmative consent (a recorded statement confirming identity). Cloned voice IDs are bound to a user account and cannot be exported.
